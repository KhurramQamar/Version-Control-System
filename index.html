A Complete Tutorial to use version control system

By Follow these steps you can grasp the concepts of version control system GIT

Following are the steps for the complete map of GIT

  1. Create Account

    Create an account on github Click Here

  2. Download Git

    Download git in your system Download

  3. Initialize

    At first Initialize the empty git repository by using git init command on your terminal

  4. Check Status

    Now check the status of your files in the folder to show the information about files by using git status Command

  5. Add Files

    Now we will add the files to staging area by using git add (filename) for a single file and git add . for adding all files

  6. Commit code

    When the files are in staging area you can save them to commit history by using command git commit -m "My First Commit" you can write any message in double quotes an check all your commit from the command git log

  7. Upload changes

    For upload changes to GitHub you have to link your local repository to github repository git remote add origin URL

  8. Push code

    To Push your code to linked repository you just have to type git push -u origin master / branch-name

  9. Pull code

    Our local repository is synchronized with online repository but if we edit code file online on the github and we want the chages to be done in local repository we have to use the git pull command it will fetch all the changed files to your local repository and now your both repositories are synchronized.

    10. Branching

  10. An exciting feature that github provides is that we can add a new branch without changing stable version by using git checkout -b (branchname) and moves the head to new branch
